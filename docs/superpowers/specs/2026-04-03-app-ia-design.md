# App Information Architecture — Curation

> 앱 전체 화면 구조, 내비게이션, 유저 플로우 설계.
> 기존 PRODUCT_PLAN 섹션 5의 탭/피드 구조를 대체한다.

---

## 1. 핵심 리텐션 루프

```
발견 ──→ 등록 ──→ 피드백 ──→ 추천 개선 ──→ 발견
 (홈)    (+/서재)   (+)       (백그라운드)    (홈)
```

서비스의 모든 화면과 기능은 이 루프를 가속하기 위해 존재한다.

---

## 2. 내비게이션 구조

### 2-1. 하단 3탭

```
┌─────────┬─────────┬─────────┐
│   홈    │    +    │   서재   │
└─────────┴─────────┴─────────┘
```

| 탭 | 역할 | 핵심 질문 |
|----|------|----------|
| **홈** | 발견 + 유도 | "오늘 뭐 읽을까?" |
| **+** | 등록 | "이 책 등록할래" |
| **서재** | 내 컬렉션 | "내가 뭘 읽었지?" |

3탭은 동일한 높이/스타일. + 탭은 FAB가 아닌 일반 탭 아이템.

### 2-2. 상단바

| 위치 | 요소 | 동작 |
|------|------|------|
| 좌 | 프로필 아바타 | 취향 화면 진입 |
| 우 | 검색 아이콘 (🔍) | 검색 화면 (홈에서만) |

### 2-3. 제거한 것

| 기존 | 이유 |
|------|------|
| ~~탐색 탭~~ | 검색은 상단 아이콘 + 등록 플로우에 내장. 별도 탭 불필요. |
| ~~추천 탭~~ | 홈 피드에 추천 섹션 통합. 분리하면 홈이 비고 추천이 중복. |
| ~~프로필 탭~~ | 상단 아바타로 진입. 자주 가는 화면이 아님. |
| ~~서재 커버 피드~~ | 서가 뷰 단독으로 충분. 홈이 큐레이션 피드 역할을 대체. |

### 2-4. 검색

홈 상단 검색 아이콘과 + 등록 플로우의 검색은 **동일한 검색 컴포넌트**를 사용한다.
- 검색 결과에서 책 탭 → 책 상세 바텀시트 (홈 진입) 또는 상태 선택 시트 (+ 진입)
- 진입 경로에 따라 결과 선택 후 동작만 다름

## 3. 화면별 설계

### 3-1. 홈

앱의 메인 진입점. Netflix처럼 큐레이션 섹션이 세로로 나열된 피드.

**상단바:** 아바타 (좌) + 검색 (우)

**피드 섹션 구성 (Phase 1):**

| 순서 | 섹션 | 조건 | 설명 |
|------|------|------|------|
| 1 | **내 책** | 읽는 중 or 피드백 미완료 책 존재 | 읽는 중 + 피드백 필요한 책을 **하나의 가로 스크롤 행**으로 통합. 각 카드에 상태별 CTA("다 읽었어요" / "피드백 남기기"). **최대 5장까지 노출**, 초과 시 "더보기". |
| 2 | 읽고싶은 책 | 원탭 저장 1권 이상 | 홈 큐레이션에서 북마크로 저장한 위시리스트 |
| 3 | 이 책은 어때요? | 추천 신뢰도 충족 | 취향 벡터 기반 추천 |
| 4+ | [테마] 큐레이션 | 읽은 책 주제 기반 | 내가 읽은 책과 비슷한 주제, **안 읽은 책만** 노출 |

**"내 책" 섹션 상세:**
- 읽는 중 카드: 표지 + 제목 + 저자 + [다 읽었어요] 버튼 → 피드백 플로우
- 피드백 필요 카드: 표지 + 제목 + 저자 + [피드백 남기기] 버튼 → 피드백 플로우
- 최대 5장까지 노출. 섹션이 무한히 길어지지 않도록.
- 향후: "안 볼래요" 스와이프/롱프레스로 개별 카드 숨기기 (피드백 안 남길 책용)

**큐레이션 카드 인터랙션:**
- 카드 탭 → 책 상세 바텀시트
- 원탭 저장 버튼 (북마크 아이콘) → "읽고싶은 책"에 추가 (토스트 피드백)
- 가로 스크롤 (섹션당 2.5~3개 표지 노출)

**Phase 2 추가 섹션:**

| 섹션 | 조건 | 설명 |
|------|------|------|
| [클러스터 라벨] 좋아하시니까 | 클러스터링 활성화 | LLM 생성 라벨 + 취향 기반 추천 |
| 나의 취향 요약 카드 | 클러스터링 활성화 | "당신은 이런 독자예요" 요약 카드 |

### 3-2. + (등록)

서비스의 핵심 액션. 탭하면 등록 플로우가 레이어로 진입.

**플로우:**

```
+ 탭 → 검색 → 책 탭 → 상태 선택 시트
                         ├─ 읽는 중 → 서재에 추가 (완료, 3탭)
                         └─ 읽었어요 → 피드백 화면 → 완료
                                        └─ "나중에 할게요" 스킵 가능
```

**상태 선택 시트:**
- 책 선택 후 바텀시트로 올라옴
- 두 개 버튼: [📖 읽는 중] / [✅ 읽었어요]
- "읽고싶은 책"은 여기 없음 — 홈 북마크로 처리

**피드백 입력 ("읽었어요" 선택 시):**

```
평가 (👍 좋았어요 / 🤔 보통이에요 / 👎 별로였어요)
  → 감정 태그 선택 (캐릭터 / 문체 / 세계관 / 스토리 / 메시지 / 분위기)
  → 한 줄 감상 (선택, 자유 텍스트)
  → [완료] 또는 [나중에 할게요]
```

- "나중에 할게요" 선택 시에도 서재에 "읽은 책"으로 등록됨. 피드백만 나중에.
- 스킵한 책은 홈 "내 책" 섹션의 "피드백 남기기" 카드로 노출.

### 3-3. 서재

내가 등록한 책 컬렉션. **서가 뷰가 기본**.

**상단:** "내 서재" + 통계(읽은 책 N / 읽는 중 N) + [+ 추가] 버튼
- [+ 추가]는 + 탭과 동일한 등록 플로우 진입

**읽는 중 섹션:**
- 서가 위 가로 스크롤 카드 행
- 각 카드: 표지 + 제목 + 저자 + [다 읽었어요] → 피드백 플로우
- 여러 권이면 가로 스크롤로 자연스럽게 처리

**서가 뷰 (기본):**
- 읽은 책을 컬러블록 책등으로 선반에 나열
- 롱프레스 → 드래그 정렬
- 마일스톤 배경 변화

**책등 텍스트 규칙:**
- 제목: 세로 쓰기(vertical-rl), 주색 블록에 배치, 넘치면 줄임표
- 저자: 보조색 블록에 배치, **4글자까지 보장**, 초과 시 줄임표
- 저자 영역 높이 최소 36px 확보

### 3-4. 책 상세 (바텀시트)

홈 큐레이션에서 책 카드를 탭하면 바텀시트로 올라옴.

| 영역 | 내용 |
|------|------|
| 상단 | 표지 + 제목 + 저자 + 한 줄 소개 |
| 액션 | [읽는 중] + [읽었어요] + [북마크] 세 버튼 |
| 비슷한 책 | book-to-book 유사도 기반 (추천 서버 /similar 엔드포인트) |

**액션별 동작:**
- [읽는 중] → 서재에 추가, 토스트 확인 (2탭 완료)
- [읽었어요] → 피드백 플로우 진입 (스킵 가능)
- [북마크] → "읽고싶은 책"에 추가

### 3-5. 취향 (상단 아바타 진입)

프로필 + 취향 분석 화면. 단순 설정이 아닌, "나를 아는" 공간.

**Phase 1:**
- 독서 통계 (등록 권수, 피드백 수, 장르 분포)
- 계정 설정, 알림 설정, 문의하기

**Phase 2:**
- 취향 요약 ("당신은 캐릭터 성장 서사를 좋아하는 독자예요")
- 키워드 태그 시각화
- 피드백 히스토리

## 4. 유저 플로우 맵

### 주요 플로우

| 플로우 | 경로 | 탭 수 |
|--------|------|-------|
| 홈→읽는 중 등록 | 홈 카드 탭 → 상세 → [읽는 중] | 2탭 |
| 홈→읽었어요 등록 | 홈 카드 탭 → 상세 → [읽었어요] → 피드백 | 2탭 + 피드백 |
| 홈→북마크 | 홈 카드 북마크 아이콘 | 1탭 |
| +→읽는 중 | + → 검색 → 책 → [읽는 중] | 3탭 |
| +→읽었어요 | + → 검색 → 책 → [읽었어요] → 피드백 | 3탭 + 피드백 |
| 서재→등록 | [+ 추가] → 검색 → 책 → 상태 선택 | 3탭 |
| 서재→다 읽었어요 | 읽는 중 카드 → [다 읽었어요] → 피드백 | 1탭 + 피드백 |
| 홈→피드백 | "내 책" 카드 → [피드백 남기기] → 피드백 | 1탭 + 피드백 |

## 5. "읽고싶은 책" (위시리스트)

기존 MVP에서 제외했던 위시리스트를 **가볍게** 부활.

**저장 방식:** `user_books.status = 'want_to_read'` (ARCHITECTURE.md 반영 완료)

**진입점:**
- 홈 큐레이션 카드의 원탭 북마크 아이콘
- 책 상세 바텀시트의 [북마크] 버튼

**노출 위치:**
- 홈 피드에 "읽고싶은 책" 섹션으로 노출

**제한:**
- 서재에는 노출하지 않음 (서재 = 서가 뷰, 읽은 책 + 읽는 중만)
- 별도 화면/탭 없음
- 피드백 없음 (아직 안 읽었으니까)

## 6. 온보딩 & 빈 상태

### 6-1. 온보딩 (신규 유저)

앱 최초 실행 시 온보딩은 **3탭 내비게이션 진입 전 풀스크린 플로우**로 진행.
온보딩 완료 후 홈 탭으로 랜딩.

상세 플로우는 PRODUCT_PLAN 섹션 4-1 정의를 따르되, IA 스펙에 맞게 조정:

```
웰컴 → 유명 책 표지 그리드에서 탭탭탭 선택 (최소 5권)
     → "제일 좋았던 책 하나만 골라볼까?"
     → 간단 피드백 (평가 + 태그)
     → "서재가 시작됐어요!" → 홈 탭 진입
```

**온보딩 결과물:**
- 선택한 책 → 서재에 "읽은 책"으로 등록
- 피드백 남긴 1권 → 추천 엔진 시드 데이터
- 나머지 4+ 권 → 홈 "내 책" 섹션에 "피드백 남기기"로 노출

### 6-2. 빈 홈 상태 (온보딩 직후 또는 데이터 부족)

온보딩 완료 직후의 홈은 아직 추천 데이터가 부족하므로:

| 섹션 | 상태 | 표시 내용 |
|------|------|----------|
| 내 책 | 온보딩에서 등록한 책 존재 | 피드백 필요 카드들 (온보딩에서 스킵한 책) |
| 읽고싶은 책 | 비어있음 | 노출 안 함 |
| 이 책은 어때요? | 추천 신뢰도 미충족 | "피드백을 더 남기면 맞춤 추천이 시작돼요" 안내 카드 |
| [테마] 큐레이션 | 데이터 부족 | 에디터 픽 또는 인기 도서 기반 큐레이션 (콜드 스타트 대응) |

**데이터 없는 완전 신규 유저 (온보딩 미완료):**
- 홈 전체가 온보딩 유도 화면: "서재를 시작해볼까요?" + 온보딩 진입 CTA
