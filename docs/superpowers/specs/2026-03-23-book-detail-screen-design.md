# 책 상세 화면 설계

## 개요

내 서재에 등록된 책의 정보 확인 + 피드백(호오, 감성태그, 자유 리뷰) 수집을 위한 화면.
취향 분석 파이프라인의 핵심 입력 지점.

## 화면 성격

- **내 서재에 있는 책 전용 화면** — 미등록 책은 이 화면에 진입하지 않음
- 유저의 핵심 플로우: 서재에 책 채우기 → 리뷰 남기기 → 취향 발견

## 진입 경로

| 경로 | 동작 |
|------|------|
| 서재 → BookSpine 탭 | 상세 화면 직접 이동 |
| 검색 → 바텀시트(상태 선택) → 등록 완료 | 검색 화면에 머묾 + 스낵바("서재에 추가됨 · 보러가기") 표시. "보러가기" 탭 시 상세 화면 이동 |

검색에서 등록 후 자동 이동하지 **않는** 이유: 여러 권 연속 등록 시 흐름이 끊기지 않도록.

## 화면 구조

위에서 아래로 4개 섹션:

### 1. 책 정보 (컴팩트)

- 표지 (작게) + 제목 + 작가
- 읽기 상태 표시 + 상태 변경 버튼 (탭 → 바텀시트: 읽는 중 / 다 읽었어요)
- 쪽수, 출판사, 줄거리 등은 표시하지 않음

### 2. 호오 평가 (3단계)

- **"이 책 어때요?"** + 좋았다 / 보통 / 별로
- 읽기 상태에 관계없이 동일 문구, 동일 구조
- 단일 선택, 다시 탭하면 변경 가능
- **자동 저장** — 탭 시 즉시 서버 반영, 변경 시 업데이트
- Phase 2: "최애 책" 마크 추가 예정 (취향 분석 가중치용)

### 3. 감성 태그

- **"이 책의 느낌은?"** + 감성 칩 나열
- 복수 선택 가능, 토글 방식
- **자동 저장** — 탭/해제 시 즉시 서버 반영
- 태그 목록은 Supabase `emotion_tag_options` 테이블에서 관리 (앱 배포 없이 수정 가능)
- 초기 태그 예시: 잔잔한, 따뜻한, 긴장감, 몰입, 여운, 유쾌한, 무거운, 서정적, 속도감, 생각할거리

### 4. 자유 텍스트 피드백

- 텍스트 입력창
  - placeholder: "이 책에 대해 자유롭게 적어보세요..."
  - 명시적 **저장 버튼**으로 저장 (자동 저장 아님)

- **글쓰기 도움 패널** (접기/펼치기)
  - 입력창 아래에 "뭘 쓸지 모르겠다면?" 링크
  - 탭하면 도움 패널 펼쳐짐 (텍스트박스 내용과 완전 독립)
  - 패널 내용:
    - **속성 칩:** [캐릭터] [문체] [전개] [분위기] [메시지] [세계관]
      - 탭하면 입력창 placeholder 힌트가 해당 주제로 변경 (기존 텍스트 보존)
    - **리플렉션 질문:** "가장 기억에 남는 장면이 있나요?" + 새로고침(↻) 버튼
      - 탭하면 질문 텍스트가 입력창 끝에 추가 삽입 (기존 텍스트 보존)
      - 질문 목록은 Supabase `reflection_prompts` 테이블에서 관리

## 상태별 화면 변화

| 유저 상태 | 화면 |
|-----------|------|
| 방금 등록 (피드백 없음) | 호오 미선택 + 감성태그 비선택 + 빈 텍스트박스 |
| 호오만 남김 | 호오 선택 표시 + 나머지 비어있음 |
| 호오 + 감성태그 남김 | 선택 상태 표시 + 텍스트박스 비어있음 |
| 전부 작성 | 호오 + 태그 + 텍스트 표시, 전부 수정 가능 |

## 검색 화면 변경사항

기존: 검색 결과 탭 → 바텀시트(상태 선택) → 등록 완료 → 스낵바("서재에 추가됨")

변경: 등록 완료 후 스낵바에 **"보러가기"** 액션 추가. 탭하면 상세 화면으로 이동.

```
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: Text('${book.title} 서재에 추가됨'),
    action: SnackBarAction(
      label: '보러가기',
      onPressed: () => /* navigate to detail */,
    ),
  ),
);
```

## 데이터 조회/저장 패턴

### Provider 구조

- **bookDetailProvider** (신규) — `user_books` + `books` 조인 데이터를 단건 조회하는 FutureProvider.family(userBookId)
- 호오/감성태그 변경 → bookDetailProvider 내 mutation 메서드로 즉시 upsert + 로컬 상태 반영
- 리뷰 텍스트 저장 → 별도 mutation (저장 버튼 탭 시)
- 변경 후 bookshelfProvider도 invalidate하여 서재 목록과 동기화

### 자동 저장 실패 처리

- 호오/감성태그 자동 저장 실패 시: UI를 이전 상태로 롤백 + 스낵바("저장 실패, 다시 시도해주세요")
- 낙관적 업데이트(optimistic update) — 탭 즉시 UI 반영, 서버 실패 시 되돌림

## DB 변경

### 컬럼 추가

`user_books` 테이블:
- `rating` text — 'good' / 'neutral' / 'bad' (nullable, 미평가 허용)
- `emotion_tags` jsonb — 감성태그 **ID 배열** (예: ["uuid-1", "uuid-2"]) (nullable)
  - 라벨 문자열이 아닌 ID로 저장 — 태그 이름 변경 시 기존 데이터 정합성 유지
- `review_text` text — 자유 텍스트 리뷰 (nullable)

### 신규 테이블

`emotion_tag_options`:
- `id` uuid PK
- `label` text NOT NULL — 표시 텍스트 ("잔잔한")
- `sort_order` int — 표시 순서
- `is_active` boolean DEFAULT true — 비활성화 지원

`reflection_prompts`:
- `id` uuid PK
- `question` text NOT NULL — "가장 기억에 남는 장면이 있나요?"
- `category` text — 속성 칩과 연결 (nullable, 범용 질문은 null)
  - 카테고리 값: 'character', 'writing_style', 'worldbuilding', 'plot', 'message', 'atmosphere'
- `is_active` boolean DEFAULT true

### 글쓰기 도움 패널 — 속성 칩 설명

속성 칩 [캐릭터] [문체] [전개] [분위기] [메시지] [세계관]은 **UI 전용 가이드**이며 유저 데이터로 저장하지 않음. placeholder 힌트 변경과 리플렉션 질문 필터링에만 사용. 기존 `FeedbackCategory` enum과 값이 동일하지만, 이 칩은 구조화된 피드백 입력이 아니라 자유 텍스트 작성을 돕는 보조 장치.

### 기존 모델 정리

- `BookFeedback` 모델 (category + sentiment + freeText) → 새 구조로 교체
  - 기존 `feedbacks` 테이블은 삭제하지 않고 보류 — Phase 2에서 LLM 자동 추출 결과 저장용으로 활용 가능
  - 현재 이 테이블을 사용하는 화면/코드 없으므로 전환 비용 없음

## Phase 2 예정 사항

- 최애 책 마크 (호오 위에 별도 토글, 취향 분석 가중치)
- 장르별 감성태그 분기 (소설 vs 비문학 등)
- 장르별 리플렉션 질문 분기
- LLM으로 자유 텍스트에서 카테고리별 sentiment 자동 추출 → feedbacks 테이블 저장
- 다른 유저 리뷰 보기 (커뮤니티)

## 기술 스택

- Flutter + Riverpod (상태 관리)
- Supabase (DB, 실시간 저장)
- 기존 Warm Ink 디자인 시스템 활용
