# Design System — Curation App

> 도서 큐레이션/추천 앱의 디자인 시스템. 확정된 결정과 Flutter에서 잡을 미확정 항목을 구분한다.

---

## 1. 핵심 감정

**"편하고 깔끔하게"**

- 부담 없이 자연스러운 (토스/당근의 편안함)
- 정돈되고 군더더기 없는 (Things 3의 깔끔함)
- 이 두 감정이 공존해야 한다

---

## 2. 확정 사항

### 2-1. 타이포그래피

| 항목 | 값 | 비고 |
|------|-----|------|
| 폰트 | Pretendard Variable | 단일 폰트. Inter 기반 한글 최적화. |
| 제목 weight | 300 (Light) | 가볍고 고급스러운 느낌. Stripe 참고. |
| 본문 weight | 400 (Regular) | |
| UI/라벨 weight | 500 (Medium) | 버튼, 내비게이션, 메타정보 |
| 강조 weight | 600 (SemiBold) | uppercase 라벨 등 |
| 제목 자간 | -0.8px ~ -1.2px | 크기에 비례해서 타이트하게 |
| 본문 자간 | 0px (기본) | |
| 본문 행간 | 1.6 ~ 1.7 | 한글 가독성을 위해 넉넉하게 |
| 주 텍스트 컬러 | #0F172A (딥 네이비) | 순수 블랙보다 미세하게 따뜻. Stripe 참고. |
| 보조 텍스트 컬러 | Flutter에서 확정 | #94A3B8 (Slate 400) 방향 |

> **참고:** 이 타이포 시스템은 앱 UI 전용. 서가 뷰의 책등 폰트(나눔명조, 고운바탕 등 7개 풀)는 별도 — PRODUCT_PLAN 섹션 5-3 참조.

**계층 예시:**
- 화면 제목: 28~32px / weight 300 / ls: -1.0px
- 섹션 제목: 15~17px / weight 500 / ls: -0.3px
- 카드 제목 (책 이름): 13~14px / weight 500 / ls: -0.15px
- 보조 텍스트 (저자): 12~13px / weight 300 / color: 보조색
- 라벨: 11px / weight 600 / uppercase / ls: 0.8~1.0px

### 2-2. 배경색

**#FFFFFF (순백)**

- 표지 색상이 가장 정확하게 보임
- 가장 깔끔한 인상
- 따뜻함은 타이포와 여백으로 만든다 (색이 아님)

### 2-3. 톤

**라이트 기반**

- 기본 모드는 라이트
- 다크 모드/마일스톤 진화는 별도 결정 필요

### 2-4. 앱 구조 (프로덕트 플랜에서 유지)

| 화면 | 역할 |
|------|------|
| 커버 피드 | Netflix식 가로 스크롤 행. 표지 이미지 중심 탐색. |
| 서가 뷰 | 컬러블록 책등. 수집 만족감. 드래그 정렬. |
| 책 상세 | 표지 + 평가(좋아요/보통/별로) + 감정 태그 + 리뷰 |
| 온보딩 | 책 선택 → 첫 피드백 → 서재 완성 |
| 추천 | 취향 벡터 기반 맞춤 추천 (신뢰도 충족 시) |

### 2-5. 디자인 원칙

1. **책 콘텐츠가 유일한 컬러 소스** — UI 크롬은 무채색(딥 네이비~그레이). 커버 피드의 표지 이미지, 서가 뷰의 컬러블록 책등 등 책에서 파생된 색만 화면에 색을 입힌다. (Glass, Letterboxd 참고)
2. **적게 보여준다** — 한 화면에 요소를 최소화. 여백으로 숨을 쉰다. 정보 밀도보다 편안함 우선.
3. **여백으로 구분** — 테두리/쉐도우보다 여백과 배경색 차이로 영역을 구분한다. (Literal 참고)
4. **단일 폰트로 계층** — weight(300/400/500/600)와 크기로만 정보 위계를 만든다.

### 2-6. 레퍼런스 (유효)

| 앱 | 가져올 것 |
|-----|----------|
| Literal Club | 세리프는 안 쓰지만, 여백 기반 구분 + 표지 중심 레이아웃 |
| Letterboxd | "콘텐츠가 유일한 컬러" 철학, 타이트 그리드 (서가 뷰) |
| Things 3 | 타이포 계층, 드래그 인터랙션, 단일 액센트 절제 |
| Stripe | weight 300 제목, 정교한 타이포, 블루 틴트 쉐도우 |
| Glass | 콘텐츠 주인공 극대화, 갤러리 간격 |
| Gentler Streak | Gentle gamification (마일스톤), 데이터 시각화 |

## 3. 미확정 사항 (Flutter에서 결정)

### 3-1. 액센트 컬러

**방향: 딥 네이비 계열 (유력)**

후보:
- `#1E293B` (Slate 800) — 모노크롬보다 따뜻, 컬러보다 절제
- `#0F172A` (Slate 900) — 더 진한 버전
- 블랙 모노크롬 `#0A0A0A` — 가장 절제. 표지만 컬러.

**결정 방법:** Flutter에서 실제 표지 이미지와 함께 CTA 버튼/선택 상태/활성 탭에 적용해보고 결정.

**고려사항:**
- 표지 색상과 충돌하지 않아야 함 (파란 표지 + 파란 액센트 = 충돌)
- 피드백 액션(좋아요/보통/별로)에는 별도 의미 컬러 사용 가능

### 3-2. 컴포넌트 형태

| 항목 | 후보 | 결정 방법 |
|------|------|----------|
| 카드 radius | 4~8px | 표지 이미지와의 조화. 너무 둥글면 표지가 잘림. |
| 버튼 radius | 8px vs 50px (pill) | CTA/태그 각각 다르게 할 수 있음 |
| 쉐도우 | 블루 틴트 vs 미니멀 vs 없음 | 배경색과의 관계에서 결정 |
| 테두리 | 있음 vs 없음 (여백만) | "여백으로 구분" 원칙에 따라 최소화 방향 |

### 3-3. 마일스톤 배경 진화

**컨셉은 유지:** 권수에 따라 서재 분위기가 변한다.

| 단계 | 방향 | 미결정 |
|------|------|--------|
| 0~9권 | 순백 #FFFFFF (확정된 배경색과 동일) | — |
| 10~29권 | 미세한 변화 | 어떤 변화? 배경색? 테두리? 서가만? 전체? |
| 30~49권 | 중간 | |
| 50~99권 | 뚜렷한 변화 | |
| 100권+ | 최종 | 다크 라이브러리? 골드 액센트? |

**결정 방법:** 기본(0~9권) 디자인을 먼저 완성한 후, 단계별 변화를 실제 앱에서 실험.

### 3-4. 서가 뷰 세부

- 선반 스타일: 라인 vs 없음 vs 기타
- 책등 크기/비율
- 마일스톤에 따른 서가 배경 변화

### 3-5. 피드 레이아웃 세부

- 표지 카드 크기 (현재 140x200px 방향이지만 실제 폰에서 확인 필요)
- 행당 보이는 표지 개수 (2.5~3개)
- 섹션 간 간격
