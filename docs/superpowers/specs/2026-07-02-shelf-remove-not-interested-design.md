# 서재에서 빼기 + "관심 없어요" 취향 신호 — 설계

날짜: 2026-07-02 · 상태: Eden 승인 (대화로 A/B 방향 확정)

## 배경 (Eden 리포트)

1. 홈 '내 책'에 잘못된 판본(화차)이 들어와 있는데 **뺄 방법이 없다**. 서재는 "내가 꾸미는" 공간(핵심가치 1)인데 꽂기만 되고 빼기가 없음 — 기본 관리 기능 부재.
2. 추천("이 책은 어때요?")에서 관심 없는 책이 계속 뜬다. **담기/관심없음 같은 추천 화면에서의 행동이 취향에 반영**되어야 함 — 피드백을 따로 안 남겨도 쓸수록 추천이 좋아지는 루프.

## A. 책 삭제 (잘못 넣은 책 지우기)

**시나리오(Eden 실사례):** 홈 '내 책'에 잘못 넣은 책(오등록/오판본)이 보이는데 지울 방법이
없다. 원하는 것은 숨김이 아니라 **그 책 기록 자체의 삭제** — 내 책/서재/어디서든 사라져야 한다.

**원칙: 이유를 묻지 않는 기본 동작.** 확인 다이얼로그 없음 — 대신 실행 취소(undo) 스낵바.
⚠️ 라벨에 '서재에서 빼기' 같은 개념어를 쓰지 않는다 — 유저 언어는 그냥 "삭제"다.

### 진입점 — 바텀시트(서재 상태 인지, PR#42)가 허브

| 노출면 | 진입 | 비고 |
|---|---|---|
| 홈 '내 책' 카드 | **길게 누르기 → 바텀시트** | 탭은 기존 CTA(피드백/다읽었어요) 유지 |
| 홈 '읽고싶은 책' 카드 | 길게 누르기 → 바텀시트 | 라벨만 '읽고 싶어요 취소' |
| 서재(서가 뷰) 책 | 탭 → 바텀시트 | 기존 탭 동작에 시트가 이미 있으면 액션만 추가 |
| 검색/추천/큐레이션의 서재 보유 책 | 탭 → 바텀시트 (기존) | |

### 동작

- 바텀시트 하단 destructive 위치에 **`이 책 삭제`** 텍스트 버튼 (서재 보유 책에만 노출.
  status=wishlist 인 책은 라벨 `읽고 싶어요 취소` — 왓챠 토글 해제 패턴)
- 탭 즉시: `user_books` 행 DELETE → bookshelfProvider 갱신(내 책/서재/읽고싶은책 전부에서
  사라짐) → `triggerRecompute` fire-and-forget (기존 패턴 — user_state 트리거가 tier/likes 재계산)
- 스낵바 `삭제했어요 · 실행 취소` — 실행 취소 탭 시 삭제 전 행 그대로 재INSERT(rating/
  emotion_tags/review_text 포함 스냅숏 복원) + 재계산 트리거
- 삭제하면 그 책은 다시 추천 후보가 될 수 있음(정상 — Goodreads 동일)

## B. "관심 없어요" + 읽고싶어요 취향 반영

### 유저 저니

```
추천/비슷한책/화제의책 카드 탭 → 바텀시트
  → (서재에 없는 책) "관심 없어요" 액션
  → 카드 즉시 목록에서 사라짐(optimistic) + 스낵바 "알겠어요, 덜 보여드릴게요"
  → 이 책은 어떤 추천에도 다시 안 뜸
  → 이런 결의 책도 점점 덜 뜸 (취향 음수 반영)
```

```
추천에서 "읽고 싶어요"(찜)
  → 읽기 전이지만 "이런 쪽이 취향"이라는 약한 긍정 신호로 반영 (신규)
```

### 데이터 (서재 비오염 원칙)

- 관심없음은 서재(user_books)가 아니라 **전용 신호 테이블** `user_book_signals`
  (user_id, book_id, signal, created_at / UNIQUE(user_id, book_id, signal) / RLS 본인만)
  — v1 signal 값은 `not_interested` 하나. 노출 로그(impressions)와 분리: 노출 없이도 마킹 가능해야 하고, 취향 정본은 컴팩트해야 재계산이 싸다.
- 앱은 Supabase 직접 write(피드백과 동일 패턴) + `triggerRecompute` + 해당 추천 카드 즉시 제거 + impressions `logAction('disliked')` 유지(노출 통계용).

### 추천 엔진 반영

| 신호 | 후보 풀 | 취향 벡터 (twostage) |
|---|---|---|
| not_interested | stage1 **완전 제외**(-999 마스킹, read_ids 와 동일) + **서빙 시 캐시된 recs 에서도 즉시 필터**(재계산 전 즉효) | desc 항 **-0.75** (별로였어요 -1.5 의 절반 — 읽고 실망한 것보다 약한 신호) |
| wishlist(읽고싶어요) | 제외(기존 유지) | desc 항 **+1.0 신규** (좋았어요 desc +3.0 대비 약함 — 아직 안 읽음) |

- recompute 가 user_book_signals 를 함께 읽고 **input_hash 에 포함** → 신호 변경 시 캐시 자동 무효화
- 가중치는 config 상수로 (조정 가능하게)

### 품질 검증 (의도된 스코어링 변경 — 동등성 아님)

- 실인덱스 오프라인 before/after 리포트: 실유저 스냅숏 + 합성 케이스로
  ①not_interested 책의 유사작 순위가 실제로 하락 ②wishlist 결 책이 상승 ③기존 top-20 이 과도하게 뒤집히지 않음(안정성) 확인 후 배포
- pytest: 신호 로딩/해시/마스킹/서빙 필터 단위 테스트

## v1 범위 밖 (기록)

- 관심없음 목록 관리(되돌리기) 화면 — 되돌리기는 당장은 불가(신호 삭제 UI 없음)
- 큐레이션 섹션 카드에의 관심없음 노출(추천 계열 3곳 먼저)
- '내 책' 피드백 유도만 끄기(서재엔 유지) — 니즈 확인되면 별도
