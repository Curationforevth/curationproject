### Step 1: Welcome Screen ✅

**컨셉:** B — 빈 서재 → 도르륵 애니메이션

- 다크 배경 (#1A1818), Warm Ink 팔레트
- 카피: "당신만의 서재를 만들어볼까요?"
- 빈 선반이 먼저 보이고, 0.5초 후 책등이 하나씩 슬라이드인으로 꽂힘
- 꽂히는 책마다 컬러가 달라서 시각적 보상
- 아래 선반은 비어있어서 "더 채울 수 있다" 암시
- CTA: "서재 만들기"

### Step 2: 책 선택 화면 ✅

**컨셉:** A — 커버 그리드 (Pinterest/Spotify 패턴)

- 표지 3열 그리드 + 장르 필터 탭 (전체/소설/에세이/자기계발/SF/...)
- 프로그레스 바: "2/5" (최소 5권 권장, 강제 아님)
- 탭 = 즉시 "읽음" 체크 (상태 선택 없음, 가볍게)
- 하단 검색바: 직접 검색도 가능
- 데이터 소스: books 테이블에서 loan_count 높은 순 (정보나루 대출 데이터), 커버 이미지 있는 책, 제목 기준 중복 제거
- 0권이어도 "건너뛰기"로 진행 가능

---

### Step 3: 첫 피드백 ✅ (부분 확정)

**3-1. 최애 책 고르기**
- 선택한 책들을 표지로 나열
- "이 중에서 제일 좋았던 책은?" → 하나 탭
- 선택한 책에 약간의 가중치 부여 (is_onboarding_favorite + 추천 가중치)
- 나중에 다른 책으로 변경 가능

**3-2. 감성태그 선택**
- 최애 책에 대해 감성태그 복수 선택
- 기존 emotion_tag_options 10종: 잔잔한, 따뜻한, 긴장감, 몰입, 여운, 유쾌한, 무거운, 서정적, 속도감, 생각할거리
- "건너뛰기" 가능

**진입 조건:** 책 1권 이상 선택 시에만. 0권(건너뛰기)이면 Step 4로 직행.

## 미정 사항 및 이유

### 1. Step 4 (서재 완성 축하) — 무엇을 보여줄 것인가

**미정 이유:** 온보딩 완료 후 유저에게 **입력의 효과를 체감시켜야** 하는데, 이를 위해선 추천 엔진이 어떤 형태로든 작동해야 함. 단순히 "서재가 완성됐어요!" 축하 메시지만으로는 유저가 피드백을 남길 동기가 생기지 않음.

**의존성:** 추천 엔진 스펙 — 온보딩 직후 어떤 추천/분석 결과를 보여줄 수 있는지에 따라 이 화면의 내용이 결정됨.

### 2. "입력할수록 추천이 좋아진다" UX

**미정 이유:** 카피(텍스트)만으로는 전달 불가. Duolingo는 목표 변경 시 UI가 즉시 반응하고, TikTok은 사용할수록 피드가 눈에 띄게 달라짐. **유저가 입력 → 결과 변화를 체감하는 인터랙션**이 필요한데, 이건 추천 엔진이 뭘 반환하는지에 따라 설계가 달라짐.

**의존성:** 추천 엔진 스펙 — 최소 몇 권/몇 개 태그가 있어야 추천 퀄리티가 보장되는지.

### 3. 빈 서재 / 데이터 부족 상태 UI

**미정 이유:** 책 0권, 1권, 3권, 10권일 때 각각 추천이 가능한지, 어느 수준인지에 따라 UI가 달라짐. "아직 3권 더 추가하면 추천을 받을 수 있어요" vs "추천 준비 완료!" 같은 프로그레스 표시는 추천 엔진의 최소 데이터 요구량이 정해져야 설계 가능.

**의존성:** 추천 엔진 스펙 — 추천 가능 최소 조건 (minimum viable recommendation).

### 4. 프로그레스 바 기준값

**미정 이유:** Step 2에서 "최소 5권 권장"이라고 했지만, 실제로 추천 퀄리티가 보장되는 최소 권수가 5권인지 10권인지 20권인지는 추천 엔진 설계에서 결정됨. 이 수치에 따라 프로그레스 바 목표치와 온보딩 카피가 바뀜.

**의존성:** 추천 엔진 스펙 — 최소 입력 권수.
