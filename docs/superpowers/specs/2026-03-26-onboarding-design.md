## 확정된 사항

### 전체 구조: 4단계

```
Welcome → 책 선택 → 첫 피드백 → 서재 완성
```

모든 단계에서 "건너뛰기" 가능. 0권 선택해도 서비스 이용 가능.

---

### Step 1: Welcome Screen ✅

**컨셉:** B — 빈 서재 → 도르륵 애니메이션

- 다크 배경 (#1A1818), Warm Ink 팔레트
- 카피: "당신만의 서재를 만들어볼까요?"
- 빈 선반이 먼저 보이고, 0.5초 후 책등이 하나씩 슬라이드인으로 꽂힘
- 꽂히는 책마다 컬러가 달라서 시각적 보상
- 아래 선반은 비어있어서 "더 채울 수 있다" 암시
- CTA: "서재 만들기"

---

### Step 2: 책 선택 화면 ✅

**컨셉:** A — 커버 그리드 (Pinterest/Spotify 패턴)

- 표지 3열 그리드 + 장르 필터 탭 (전체/소설/에세이/자기계발/SF/...)
- 프로그레스 바: "2/5" (최소 5권 권장, 강제 아님)
- 탭 = 즉시 "읽음" 체크 (상태 선택 없음, 가볍게)
- 하단 검색바: 직접 검색도 가능
- 데이터 소스: books 테이블에서 sales_point 높은 순, 커버 이미지 있는 책
- 0권이어도 "건너뛰기"로 진행 가능

---

### Step 3: 첫 피드백 ✅ (부분 확정)

**3-1. 최애 책 고르기**
- 선택한 책들을 표지로 나열
- "이 중에서 제일 좋았던 책은?" → 하나 탭
- 선택한 책에 약간의 가중치 부여 (is_onboarding_favorite + 추천 가중치)
- 나중에 다른 책으로 변경 가능

**3-2. 감성태그 선택**
- 최애 책에 대해 감성태그 복수 선택
- 기존 emotion_tag_options 10종: 잔잔한, 따뜻한, 긴장감, 몰입, 여운, 유쾌한, 무거운, 서정적, 속도감, 생각할거리
- "건너뛰기" 가능

**진입 조건:** 책 1권 이상 선택 시에만. 0권(건너뛰기)이면 Step 4로 직행.

---

### Step 4: 서재 완성 ✅

**온보딩 완료 → 서재 메인 진입. 별도 축하 화면 없이, 서재 자체가 보상.**

온보딩 직후 서재에서 보이는 것:
- 유저가 선택한 책들이 이미 서재에 꽂혀있음 (서가 뷰 애니메이션)
- **"비슷한 책" 추천이 즉시 작동** — 책 상세 들어가면 바로 유사 책 추천
- 신뢰도 스코어 충족 시 → 서재 메인에 "추천 섹션" 행 활성화
- 미충족 시 → 추천 프로그레스 안내 ("감성태그를 남기면 추천이 더 정확해져요")

> 핵심: 축하 메시지가 아니라 **작동하는 추천이 보상**. 유저가 "오, 내가 고른 책 기반으로 바로 추천이 나오네?" 체감.

---

### 빈 서재 / Empty State ✅

추천 신뢰도 스코어에 따라 UI가 자동 분기:

| 유저 상태 | 추천 상태 | 서재 메인에 보이는 것 |
|---|---|---|
| **0권 (건너뛰기)** | 추천 불가 | 빈 서재 + "책을 추가해보세요" CTA + 검색 유도 |
| **1~N권, 신뢰도 미달** | book-to-book만 | 내 책들 + "비슷한 책"(상세에서) + 프로그레스 안내 |
| **신뢰도 충족** | 취향 추천 활성화 | 내 책들 + **추천 섹션 행** + "비슷한 책" |

**프로그레스 안내 (신뢰도 미달 시):**
- 신뢰도 스코어를 기반으로 가장 효과적인 다음 행동을 안내
- "감성태그를 남기면 추천이 더 정확해져요" (피드백 깊이가 낮을 때)
- "다른 장르 책도 추가해보세요" (장르 다양성이 낮을 때)
- 프로그레스 바/수치는 신뢰도 스코어를 시각화 (정확한 표현은 실험 후 결정)

> 고정된 "N권 더 추가하세요"가 아니라, 신뢰도 스코어의 약한 축을 기반으로 동적 안내.

---

### "입력 → 추천 변화" 체감 UX ✅

추천 엔진의 즉시 경로(RPC)가 이를 해결:

1. 유저가 피드백 제출 (rating + 감성태그)
2. RPC로 취향 벡터 즉시 재계산
3. 서재 돌아오면 추천 섹션이 갱신되어 있음
4. 새 피드백 직후 "추천이 업데이트됐어요" 시각적 표시 (뱃지 또는 하이라이트)

---

### 프로그레스 바 (Step 2) ✅

온보딩 책 선택 화면의 프로그레스 바 기준:
- 고정 권수가 아니라, 신뢰도 스코어 기반으로 표시
- 다만 온보딩 시점에서는 아직 피드백 없이 책만 선택하는 단계이므, 단순히 "선택한 책 수"를 표시
- "5권 권장"은 UX 가이드라인으로 유지하되 강제 아님 (0권도 진행 가능)
- 실제 추천 활성화 기준은 신뢰도 스코어가 결정 (온보딩 이후)
