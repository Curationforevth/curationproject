# Curation Project — Product Plan

> 컨텐츠 소비 피드백 기반 취향 분석 & 추천 앱
> MVP: 도서 카테고리로 시작

---

## 1. 프로덕트 비전

**"베스트셀러만 읽는 시대에, 나만의 취향을 찾아가는 즐거움"**

사용자가 읽은 책에 대한 피드백을 남기면, 취향을 분석하고 취향에 맞는 새로운 책을 추천해주는 앱.

- 장기 목표: 도서 → 영화, 뮤지컬, 전시 등 전 카테고리 확장
- 타겟: 자기 취향을 아직 잘 모르는 일반 유저
- 플랫폼: iOS + Android 동시 출시

---

## 2. 핵심 가치

| # | 가치 | 설명 |
|---|------|------|
| 1 | **서재 경험** | 책을 하나씩 꽂아가는 뿌듯함. "한 권 더 꽂고 싶다"는 리텐션 루프 |
| 2 | **취향 발견** | 쌓인 피드백으로 "나는 이런 독자구나"를 알게 되는 경험 |
| 3 | **맞춤 추천** | 베스트셀러가 아닌, 나의 취향에 맞는 책 발견 |

> 서재는 추천을 위한 데이터 수집 수단이지만, 그 자체로 다른 서재 앱과 차별화된 감성적 경험을 제공해야 한다.

---

## 3. 로드맵

### Phase 1 — 서재 + 피드백 수집 (MVP)

- 책 검색 & 등록 (알라딘 API / Google Books API)
- 감성 서재 UI (책등 디자인, 꽂히는 애니메이션, 마일스톤)
- 피드백 수집 (가이드 질문 + 자유 텍스트)
- 온보딩 플로우

### Phase 2 — 취향 프로필

- 쌓인 피드백을 LLM으로 분석하여 "취향 요약" 제공
- 예: "당신은 캐릭터 성장 서사를 좋아하는 독자예요"
- 추천 없이도 유저에게 가치 있는 단계

### Phase 3 — 추천 엔진

- 벡터 유사도 기반 도서 추천
- 유저 피드백 + 책 메타데이터 임베딩
- n권 이상 피드백 쌓인 유저에게 추천 시작

## 4. MVP 핵심 유저 플로우

### 4-1. 온보딩 (목표: 3분 이내)

```
유명 책 표지 그리드 노출
   → "이 중에 읽은 책 있어?" 탭탭탭 선택
   → 선택할 때마다 서재에 꽂히는 애니메이션
   → 5권 이상 선택 시
   → "제일 좋았던 책 하나만 골라볼까?"
   → 가이드 질문 1~2개로 간단한 피드백
   → "당신의 서재가 시작됐어요" + 서재 화면
```

레퍼런스:
- **Spotify** — 아티스트 선택 시 버블 커지는 애니메이션, 선택의 쾌감
- **Pinterest** — 관심사 탭탭탭, 실시간 피드 변화
- **Letterboxd** — 영화 포스터 기반 별점 온보딩, 시각적 풍성함
- **Duolingo** — 온보딩 직후 "첫 수업 완료!" 즉시 성취감

### 4-2. 책 등록

```
책 검색 (제목/저자)
   → DB 매칭 (알라딘/Google Books)
   → 서재에 추가 (책등 애니메이션)
   → 피드백 입력 유도
```

### 4-3. 피드백 입력

```
"이 책에서 가장 기억에 남는 건?" (자유 텍스트)
   → "어떤 점이 좋았어?" 선택지:
      캐릭터 / 문체 / 세계관 / 스토리 전개 / 메시지 / 분위기
   → 선택 후 "구체적으로?" (자유 텍스트)
   → "별로였던 점은?" (동일 구조, optional)
```

> 좋았던 것 + 별로였던 것 모두 수집. 별로였던 피드백은 negative signal로 활용.

## 5. 서재 UI 설계

### 차별화 포인트

| 요소 | 설명 |
|------|------|
| **책등 자동 생성** | 표지 색상 추출 → 제목+저자 세로 배치. 유저만의 컬러 팔레트 |
| **꽂히는 애니메이션** | 책 추가 시 "톡" 하고 꽂히는 모션 |
| **서재 성장** | 10권 → 50권 → 100권 마일스톤마다 서재 배경 업그레이드 (작은 방 → 서재 → 도서관) |
| **읽는 중 / 읽고 싶은 책** | 서재가 빨리 차 보이도록. 상태별 시각 구분 |

### 무드보드 키워드

- warm wood tones, cozy library aesthetic
- minimal book spine design
- soft ambient lighting
- progress visualization
- 참고 검색: Dribbble "bookshelf app UI", Pinterest "cozy library illustration"

## 6. 취향 모델링 — 벡터 유사도 방식

### 왜 이 방식인가

기존 방식의 한계:
- 책의 특징(태그)을 사람이 미리 정의해야 함
- 새로운 피드백 유형이 생기면 분류 체계 수정 필요
- 취향 간 correlation 공식을 직접 설계해야 함

벡터 유사도가 해결하는 것:
- 분류 체계 없이 자연어를 그대로 벡터(숫자)로 변환
- "캐릭터 성장이 좋았어" ≈ "주인공의 변화가 인상적" → 자동으로 유사한 벡터
- 취향 간 관계도 벡터 공간에서 자연스럽게 형성

### 구조

```
[유저 피드백]              [책 메타데이터]
자유 텍스트 + 가이드 응답    줄거리 + 리뷰 + 장르 + 키워드
         ↓                          ↓
     LLM embedding              LLM embedding
         ↓                          ↓
     유저 취향 벡터              책 특징 벡터
              ↓        ↓
           코사인 유사도 계산
                ↓
          "이 책 어때요?"
```

### 설계 포인트

- **취향 벡터는 여러 개**: 한 사람이 SF도 에세이도 좋아할 수 있음. 클러스터링으로 취향 축 분리
- **피드백 품질 확보**: 가이드 질문으로 구조화된 데이터 + 자유 텍스트 조합
- **Negative signal 활용**: "별로였던 점"으로 해당 방향의 책 필터링

### 콜드 스타트 대응

- 온보딩에서 빠르게 5~10권 등록 유도
- 5권 + 피드백 3개부터 초기 추천 시작 (정확도 < 시작했다는 경험)
- 추천이 시작되기 전에도 "취향 요약"으로 가치 제공

## 7. 기술 스택 (제안)

| 영역 | 스택 | 이유 |
|------|------|------|
| **앱 (프론트)** | Flutter | iOS+Android 크로스플랫폼, 서재 애니메이션 구현에 강함, 코드베이스 하나로 효율적 |
| **백엔드 인프라** | Supabase | 인증/DB/스토리지 올인원, 초기 서버 관리 부담 없음 |
| **책 데이터** | 카카오 책 검색 API (메인) + 알라딘 API (보완) | 카카오: 실시간 검색, 알라딘: 베스트셀러 배치 로드 |
| **임베딩** | OpenAI text-embedding-3-small | 비용 저렴, 품질 충분 |
| **벡터 저장** | Supabase pgvector | 별도 벡터 DB 없이 Supabase 내에서 해결 |
| **추천 로직** | Python (FastAPI) | 벡터 계산 + 추천 알고리즘 생태계 최적 |

> 이 스택은 제안이며, 개발자 리뷰 후 확정 필요.
